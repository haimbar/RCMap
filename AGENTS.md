# AGENTS.md — RCMap Developer Reference

This file is for AI coding assistants. It describes the RCMap R package in
enough detail to make substantive contributions without human hand-holding.

---

## 1. Project Overview

RCMap is an open-source implementation of **group concept mapping** (GCM) in R
(GPL-2). GCM is a mixed-methods participatory research technique in which
stakeholders:

1. **Sort** a set of statements (cards) into thematic piles.
2. **Rate** each statement on one or more Likert-scale criteria (e.g.
   importance, feasibility).
3. A researcher then builds a shared **concept map** — a 2-D MDS projection of
   how similarly the group sorted the cards — and clusters the map into named
   thematic regions.

The package provides `RCMapMenu()`, a fully interactive text-based
menu-driven interface that walks users through the entire pipeline without them
needing to write any R code. All analysis parameters are exposed through an
auto-generated `config.txt` file in the project folder.

**Current version:** 0.3.4
**Package URL:** https://haimbar.github.io/RCMap/
**Issue tracker:** https://github.com/haimbar/RCMap/issues

## 3. Architecture and Design Decisions

### 3.1 Global State: `cmapdat`

The entire session state is stored in a single named list `cmapdat` that lives
in the user's global environment. Nearly every function reads from or writes to
`cmapdat` directly. The list is populated by `initCMap()` and persisted to
`CMapSession.RData` in the project folder.

`globals.R` declares `cmapdat` as a known global via
`utils::globalVariables(names="cmapdat", package="RCMap")` to suppress R CMD
CHECK NOTEs about undefined variables. The commented-out
`#cmapdat <<- NULL` line is a vestige; do not uncomment it.

### 3.2 Menu-Driven UI via `RCMapMenu()`

`RCMapMenu()` is a single `while(menuLevel >= 0)` loop. `menuLevel` acts as a
state variable:

- `menuLevel == 0` → render the main menu
- `menuLevel == 1` → stay in the current sub-menu after an action
- `menuLevel == -1` → exit the loop (return to R prompt)

The main menu has seven entries (indices 1–7):

| Index | Label | Action |
|-------|-------|--------|
| 1 | Choose the data folder | Calls `loadRCMapData()` → `initCMap()` |
| 2 | Summary | Split-half and Jaccard stability plots |
| 3 | Settings | Distance metric, cluster count, cluster names, color scheme, label editor |
| 4 | Plots | 12 plot types (MDS, cluster, dendrogram, rating plots, etc.) |
| 5 | Reports | Sorter / rater / statement CSV reports |
| 6 | Analysis | One-way ANOVA and Tukey HSD |
| 7 / 0 | R prompt | Exit loop |

Menu items 2–6 are shown in `silver()` (dimmed) until data is loaded; the
guard is `show_all <- !is.null(cmapdat$x)`.

### 3.3 `config.txt` System

On the first call to `initCMap()`, a `config.txt` is created in the project
folder. On subsequent calls, any missing keys are appended with defaults; keys
already present are never overwritten. Supported keys and their defaults:

| Key | Default | Type |
|-----|---------|------|
| `ratingscale` | `5` | numeric |
| `clust_method` | `ward.D2` | character |
| `dist_metric` | `Euclidean` | character |
| `color_scheme` | `rcmap` | character |
| `n_clusters` | `3` | integer |
| `mds_seed` | `154204` | integer |
| `splithalf_seed` | `23456` | integer |
| `splithalf_B` | `20` | integer |
| `fuzzy_label_threshold` | `0.15` | numeric |
| `jaccard_threshold` | `0.3` | numeric |

Config parsing is done inline in `initCMap()` using `readLines()` + regex.
Comment lines (starting with `#`) and blank lines are skipped. The internal
helper `cfg_get(key)` extracts typed scalars from the merged config map.

### 3.4 The Concept Mapping Pipeline (inside `initCMap()`)

1. **Read and validate** `Statements.csv`, `SortedCards.csv`, `Ratings.csv`,
   `Demographics.csv` via `read_statements()`, `read_piles()`,
   `read_ratings()`, `read_demographics()`.
2. **Fuzzy label matching**: pile labels from all sorters are clustered using
   Jaro-Winkler string distance (`stringdist::stringdistmatrix()` with
   `method="jw", p=0.1`) and hierarchical clustering at threshold
   `fuzzy_label_threshold`. Each cluster of similar labels is collapsed to the
   most frequent lowercase form (the canonical label). The mapping is stored in
   `cmapdat$label_dict` and saved to `dictionaries.RData`.
3. **Adjacency matrices**: `getAdjMatrices()` builds one SxS binary matrix per
   sorter where cell (i,j) = 1 if cards i and j were placed in the same pile.
4. **Optional weighting**: if `Weights.csv` exists, adjacency matrices are
   scaled by normalised weights before summing.
5. **Distance matrix**: `distanceMatrix()` sums adjacency matrices into a
   similarity matrix M, then computes dissimilarity as `n*(J-I) - M` with a
   small jitter for numerical stability (`set.seed(mds_seed)` is called here).
6. **MDS**: `smacof::mds()` projects into 2-D. Coordinates stored as
   `cmapdat$x` and `cmapdat$y`.
7. **Distance matrices in 2-D**: Euclidean (`D2e`) via `dist()` on MDS
   coordinates; hyperbolic (`D2h`) via `diskDist()` (Poincaré metric).
8. **Jaccard stability analysis (leave-one-out)**: For every k from 2 to
   `floor(M/4)`, and for every sorter j, `clusterings()` is called with `j`
   dropped, and the Jaccard index between the full-data clustering and the
   leave-one-out clustering is computed per statement. Results stored in
   `eta_ijk_e` and `eta_ijk_h`.
9. **Save session**: `cmapdat` written to `CMapSession.RData`; label
   dictionaries to `dictionaries.RData`.
10. **`update_clusters()`** is called by `loadRCMapData()` after `initCMap()`
    returns to populate `cmapdat$clusMember`.

### 3.5 Session Resumption

If `CMapSession.RData` exists in the project folder, `initCMap()` prompts the
user to either reuse it (press Enter) or reload from raw CSVs (type `L`). If
reused, the function loads and returns `cmapdat` immediately, skipping all
computation. `dictionaries.RData` (containing `card_name_dict`, `sorters_dict`,
`label_dict`) is also reloaded if present.

## 4. Key Data Structures

### 4.1 `cmapdat` — the Global Session List

Populated by `initCMap()` and stored as the named list below. All plotting and
reporting functions access it via `with(cmapdat, { ... })` or direct
`cmapdat$field` references.

| Field | Type | Description |
|-------|------|-------------|
| `n.indiv` | integer(1) | Number of sorters (N) |
| `cardNames` | data.frame | Statements: columns `StatementID` (numeric), `Statement` (character), `SeqID` (integer), `LC` (lowercase text) |
| `adj.mat` | list[N] | Per-sorter SxS numeric adjacency matrices (possibly weighted) |
| `DS` | matrix SxS | Dissimilarity matrix; rownames = `StatementID` values |
| `D2e` | matrix SxS | Euclidean 2-D distances from MDS projection |
| `D2h` | matrix SxS | Hyperbolic (Poincaré) 2-D distances from MDS projection |
| `x`, `y` | named numeric[S] | MDS coordinates; names = `StatementID` values |
| `stress` | numeric(1) | MDS stress value from `smacof::mds()` |
| `ratings` | data.frame | `RaterID` (character), `StatementID` (factor), rating columns 3..p |
| `demographics` | data.frame | Raw demographics, first column `RaterID` |
| `ratingDemographics` | data.frame | Cleaned demographic variables (no ID column), aligned to `ratings` rows |
| `cohorts` | data.frame | Binary indicator columns (one per subgroup); first column always `allRaters` = 1 |
| `eta_ijk_e` | list[max_nc-1] | Jaccard index arrays for Euclidean metric; `[[k-1]]` = k-cluster solution |
| `eta_ijk_h` | list[max_nc-1] | Jaccard index arrays for hyperbolic metric; same indexing |
| `pileLabels` | data.frame | Columns `pileLabel` (canonical), `cardNum` (integer) |
| `issues` | character(1) | Accumulated warning strings from `getAdjMatrices()` |
| `clustname` | named list | User-assigned cluster names; keys like `"clust_03_01"` (zero-padded n_clusters _ cluster_index) |
| `clustMethod` | character(1) | hclust method, e.g. `"ward.D2"` |
| `distmetric` | character(1) | `"Euclidean"` or `"Hyperbolic"` |
| `nclust` | integer(1) | Current number of clusters |
| `clusMember` | named integer[S] | Cluster assignment per statement; populated by `update_clusters()` |
| `clrscm` | character(1) | Color scheme: `"rcmap"` or `"rainbow"` |
| `cohortCol` | integer(1) | Column index into `cohorts` for the currently active subgroup |
| `pcoordChoice` | character vector | Selected `"RatingVar/CohortCol"` combinations for parallel coords / Go-Zone |
| `splhalf` | list or list() | Result of `splitHalf()`: `$cors` (numeric[B]) and `$distmetric` |
| `ratingscale` | numeric(1) | Maximum Likert value (from config.txt) |
| `dataDir` | character(1) | Absolute path to project folder, with trailing `/` |
| `sorters_dict` | data.frame | Columns `orig_ID`, `seq_ID`; maps original sorter IDs to sequential indices |
| `label_dict` | data.frame | Columns `pileLabels`, `lcpilelabels`, `canonical` |
| `mds_seed` | integer(1) | Seed used for MDS jitter in `distanceMatrix()` |
| `splithalf_seed` | integer(1) | Base seed for `splitHalf()` |
| `splithalf_B` | integer(1) | Replication count for `splitHalf()` |
| `jaccard_threshold` | numeric(1) | Threshold for flagging unstable statements in `plotjaccard()` |

### 4.2 Input CSV Formats

**`Statements.csv`** — two columns, with a header row:
```
StatementID,Statement
1,We need better training for staff
2,Funding should be increased
```
- `StatementID`: unique numeric integers (need not be consecutive).
- `Statement`: free text.

**`SortedCards.csv`** — no header row; variable number of columns:
```
sorterID, pileLabel, card1, card2, card3, ...
101,Health,3,7,12
101,Education,1,4
102,,2,5,8
```
- Column 1: sorter ID (any unique identifier).
- Column 2: pile label (text, may be blank).
- Columns 3+: card numbers matching `StatementID` values.
- One row per pile; a sorter with M piles has M rows.

**`Ratings.csv`** — header row required:
```
RaterID,StatementID,Importance,Feasibility
101,1,4,3
101,2,5,2
```
- Column 1: must be named `RaterID`, `UserID`, or `SorterID`; renamed to `RaterID`.
- Column 2: must be named `StatementID`; values must match `Statements.csv`.
- Columns 3+: numeric Likert ratings in `[1, ratingscale]`.
- One row per rater-statement combination.

**`Demographics.csv`** — header row required:
```
RaterID,Age,Gender
101,34,F
102,28,M
```
- Column 1: must be named `RaterID`, `UserID`, or `SorterID`.
- Remaining columns: demographic variables.
  - Factors → one-hot encoded into binary cohort indicators.
  - Numerics → split at median into `_H`/`_L` binary indicators.
  - Columns with no variation or all-distinct values are silently dropped.

**`Weights.csv`** (optional) — one row per sorter, must contain a `Weight` column:
```
Weight
1.2
0.8
1.0
```
Weights are normalised to sum to N: `wgts <- n.indiv * wgt_vals / sum(wgt_vals)`.

**`config.txt`** (optional) — plain text, auto-generated, `key=value` pairs:
```
# ratingscale: number of Likert scale points (e.g. 5, 7)
ratingscale=5

# n_clusters: initial number of clusters
n_clusters=4
```

### 4.3 Jaccard Index Arrays (`eta_ijk_e` / `eta_ijk_h`)

Both are R lists of length `max_nc - 1`, where `max_nc = floor(S / 4)` and S
is the number of statements. **The critical indexing rule:**

```
eta_ijk_e[[k-1]]  →  k-cluster solution
```

So `[[1]]` = 2-cluster, `[[2]]` = 3-cluster, etc. Each element is an N × S
numeric matrix where `[j, i]` is the Jaccard index for statement i when sorter
j is left out and the k-cluster solution is used.

### 4.4 Cluster Name Keys in `cmapdat$clustname`

Names are stored as a named list keyed by strings of the form:

```r
key <- sprintf("clust_%02d_%02d", nclust, clusterIndex)
# e.g. "clust_03_01" means cluster 1 in a 3-cluster solution
```

`clusterNames()` iterates `cl` from 1 to `cmapdat$nclust`, builds the key, and
returns the stored name or falls back to `as.character(cl)`.

## 5. Development Workflow

### 5.1 Regenerating Documentation

All documentation is in roxygen2 blocks inside `R/RCMap.R`. After editing any
`#'` block or the `@export` / `@importFrom` tags:

```r
# From the package root:
roxygen2::roxygenise()
```

This rewrites `NAMESPACE` and all files under `man/`. Never edit those files
directly.

### 5.2 Building and Checking the Package

```bash
# From the parent directory of RCMap/:
R CMD build RCMap
R CMD check --no-tests RCMap_*.tar.gz
```

The `--no-tests` flag is needed because there is no `tests/` directory; without
it `R CMD check` does not error, but there are no tests to run regardless.

### 5.3 Knitting the User Guide

```r
rmarkdown::render('Documentation.Rmd')
```

`Documentation.Rmd` is in the package root and is not a vignette. It renders
to HTML and is served via GitHub Pages (pkgdown). It uses figures from the
`Figures/` directory.

### 5.4 Installing Locally for Testing

```r
# From inside the RCMap/ directory:
devtools::load_all()   # fast interactive reload
# or
devtools::install()    # full install
```

## 6. Coding Conventions

### 6.1 Naming

- **Exported functions**: `camelCase` with a capital first letter:
  `RCMapMenu`, `showMDSPlot`, `showClusterPlot`, `splitHalf`, `MPindex`,
  `clusterANOVA`, `clusterTUKEY`, `distanceMatrix`, `getAdjMatrices`.
- **Internal/helper functions**: `camelCase` lowercase-first:
  `initCMap`, `loadRCMapData`, `update_clusters`, `clusterConfig`,
  `clusterNames`, `clusterCols`, `colLab`, `topLine`, `toText`, `rcmenu`,
  `clusterings`, `get_cmat`, `choose_dir`, `getOS`.
- **Read/validate functions**: `snake_case`: `read_statements`, `read_piles`,
  `read_ratings`, `read_demographics`.
- **Parameters**: `camelCase` or `snake_case` depending on context; no strict
  rule. Examples: `adjMats`, `clustMethod`, `disttype`, `max_nc`,
  `fuzzy_label_threshold`.

### 6.2 Global Assignment with `<<-`

`<<-` is used deliberately and pervasively for updating `cmapdat` from inside
`with()` blocks and from menu sub-routines. Examples:

```r
cmapdat$distmetric <<- toText("distance", dtype)
cmapdat$nclust    <<- tmpnclust
cmapdat$clusMember <<- clus
clustMethod        <<- clustMethod   # also sets a bare global in initCMap
```

This is an intentional design choice — the whole package is built around a
mutable global session object. Do not attempt to refactor this to functional
style without understanding that every menu branch depends on it.

### 6.3 `with(cmapdat, { ... })`

Most plotting and reporting functions use `with(cmapdat, { ... })` to bring
all fields into scope. Inside such blocks, `cmapdat$field <<- value` must be
used (not `field <<- value`) to write back to the global `cmapdat`.

### 6.4 Error Messages and Troubleshooting

User-facing error messages are constructed from the `troubleshooting` named
character vector defined at the bottom of `R/RCMap.R`. When adding new file
validation, extend that vector rather than inlining long strings.

### 6.5 Console Output Styling

All console output uses `crayon`: `red()`, `green()`, `yellow()`, `blue()`,
`cyan()`, `magenta()`, `bold()`, `italic()`, `underline()`, `silver()`. The
`%+%` operator is the crayon string concatenation operator. Menu items that are
unavailable (before data is loaded) are shown in `silver()`.

## 7. Common Pitfalls

### 7.1 The `[[k-1]]` Indexing for Jaccard Arrays

`eta_ijk_e` and `eta_ijk_h` use **1-based R indexing with a -1 offset**:
element `[[k-1]]` holds data for the k-cluster solution. Accessing
`[[nclust-1]]` is correct; using `[[nclust]]` or `[[nclust+1]]` is wrong.
This was the bug fixed in v0.3.4 in `statementReport()`. Every function that
reads these arrays (`showMDSPlot`, `statementReport`, `plotjaccard` via the
menu) must use `[[nclust-1]]`.

### 7.2 Non-Exported Functions with Man Pages

Several internal functions are documented with roxygen2 but intentionally NOT
listed in `NAMESPACE` (no `@export` tag):

- `sumsq`, `get_cmat`, `clusterings`, `read_piles`, `read_ratings`,
  `read_demographics`, `initCMap`, `topLine`, `toText`, `colLab`, `rcmenu`,
  `clusterConfig`, `clusterNames`, `update_clusters`, `loadRCMapData`,
  `choose_dir`, `getOS`.

Their `.Rd` files exist in `man/` but they cannot be called as
`RCMap::functionName()`. This is intentional for documentation completeness. Do
not add `@export` to these unless there is a deliberate reason to expose them.

### 7.3 The `metric == 0` Early-Return Pattern

Several `show*` functions guard against "no rating variable selected" by
checking `if(metric == 0) return(NULL)` at the top. This is the signal from
`utils::menu()` that the user pressed 0 (or escaped). Similarly,
`showMDSPlot()` uses `if(metric == 0) return(NULL)` but also
`if(is.null(metric))` for the plain MDS case. Do not conflate `NULL` and `0`
in these functions.

### 7.4 `dataDir` Always Has a Trailing `/`

`initCMap()` receives `dataDir` from `loadRCMapData()` as
`paste0(choose_directory(), "/")`. All file paths are built as
`paste0(dataDir, "filename")` (no `/` between). If you ever change how
`dataDir` is set, ensure the trailing slash is preserved or all path
constructions will break.

### 7.5 `distanceMatrix()` Sets the Global RNG Seed

`distanceMatrix()` calls `set.seed(seed)` before applying `jitter()`. This
affects the global RNG state of the user's R session. This is documented but
unavoidable given the reproducibility requirement.

### 7.6 Cluster Method Is Set as a Bare Global

In `initCMap()`, the line:
```r
clustMethod <<- clustMethod
```
writes `clustMethod` as a bare variable in the calling environment (not inside
`cmapdat`). It is also stored in `cmapdat$clustMethod`. Both are used in
different places; keep them in sync.

### 7.7 Ratings Are Multiplied by Weights In-Place

After weighting the adjacency matrices, `initCMap()` also multiplies the
ratings rows for each sorter by their weight:
```r
ratings[rownums, -c(1,2)] <- ratings[rownums, -c(1,2)] * wgts[i]
```
This means `cmapdat$ratings` contains pre-weighted rating values, not raw
Likert scores. All downstream analyses (ANOVA, reports, plots) use these
weighted values.

### 7.8 `StatementID` as Factor vs. Numeric

In `read_ratings()`, `ratings[,2]` is coerced to `factor` before returning.
This makes `StatementID` a factor throughout the session. In `ratingSummary()`,
`droplevels()` is called to remove unused levels. Code that compares
`StatementID` to numeric values must coerce appropriately.

### 7.9 Screen Clearing

`topLine()` clears the screen. The behavior is controlled by the R option
`rcmap_clear_screen` (set by `RCMapMenu()`). On non-Windows systems it writes
`\033[2J\033[H` (ANSI clear) followed by `\014` (form-feed for RStudio). To
disable, call `RCMapMenu(clear_screen = FALSE)`.

## 8. Testing

There is **no test infrastructure** in this package. The `tests/` directory
does not exist and there are no `testthat` or other test files.

Manual testing procedure:
1. Prepare a project folder with the four required CSV files.
2. `library(RCMap); RCMapMenu()` and exercise all menu paths.
3. Check that `CMapSession.RData` is written and can be reloaded.
4. Run `R CMD check --no-tests RCMap_*.tar.gz` to catch documentation and
   namespace issues.

If adding automated tests in the future, place them under `tests/testthat/`
and add `testthat` to `Suggests` in `DESCRIPTION`.

Base R packages used via `importFrom` (not in `Imports` field but imported
explicitly): `graphics`, `grDevices`, `stats`, `utils`.
